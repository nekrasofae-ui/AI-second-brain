# Hypothesis System

Система стратегического планирования на основе метода «Карта гипотез» (Byndyusoft).

## Структура карты гипотез

```
Цель (Goal)
└── Субъект (Subject) — кто влияет на достижение цели
    └── Гипотеза (Hypothesis) — что изменит поведение субъекта
        └── Метрика (Metric) — как измерить результат
            └── Задача (Task) — конкретное действие для проверки
```

## Карта 1: Повышение до 320 т.р./мес

### Цель
Получить повышение до 320 т.р./мес к Q4 2026

### Субъекты

**Субъект 1: Руководство (Ярослав, Синельников)**
- Принимают решение о повышении
- Оценивают результаты работы отдела
- Смотрят на экономию времени и денег

**Субъект 2: Сотрудники отдела ПТО (6 чел + 3 + 2 удалённо)**
- Задают вопросы (30% моего времени)
- Выполняют задачи по объектам
- Используют (или не используют) инструменты

### Гипотезы

**H1.1: Инструкции снизят вопросы**
- ЕСЛИ создам базу инструкций для типовых задач
- ТО сотрудники будут находить ответы сами
- ПОТОМУ ЧТО у них будет понятный источник информации
- РЕЗУЛЬТАТ: вопросы ко мне снизятся с 30% до 10% времени

Метрики:
- Количество вопросов в день (до/после)
- Время на ответы (часов в неделю)
- % вопросов, на которые есть инструкция

Задачи:
- [ ] Собрать топ-20 частых вопросов
- [ ] Написать инструкции на топ-10
- [ ] Внедрить базу знаний, обучить команду
- [ ] Замерить количество вопросов через месяц

---

**H1.2: Программа ПТО ускорит работу**
- ЕСЛИ внедрю программу для инженеров ПТО с ИИ-ассистентом
- ТО сотрудники будут выполнять задачи быстрее
- ПОТОМУ ЧТО рутина будет автоматизирована
- РЕЗУЛЬТАТ: производительность отдела вырастет на 20-30%

Метрики:
- Время на оформление АОСР (до/после)
- Количество обработанных объектов в месяц
- Ошибки в документации (до/после)

Задачи:
- [ ] Завершить MVP программы (январь 2026)
- [ ] Протестировать на 1-2 сотрудниках
- [ ] Собрать обратную связь, доработать
- [ ] Внедрить на весь отдел
- [ ] Замерить метрики через 2 месяца

---

**H1.3: Презентация покажет ценность**
- ЕСЛИ подготовлю презентацию с конкретными результатами оптимизации
- ТО руководство увидит мою ценность в цифрах
- ПОТОМУ ЧТО они принимают решения на основе данных
- РЕЗУЛЬТАТ: согласуют повышение до 320 т.р.

Метрики:
- Экономия времени команды (часов/месяц)
- Экономия денег компании (рублей/месяц)
- Снижение ошибок (%)

Задачи:
- [ ] Собрать данные по оптимизации за 6 месяцев
- [ ] Посчитать экономический эффект
- [ ] Подготовить презентацию (август-сентябрь)
- [ ] Назначить встречу с руководством

## Карта 2: Телеграм-канал 600 подписчиков

### Цель
Вырастить канал «Где исполнительная, Лебовски?» до 600 подписчиков к концу 2026

### Субъекты

**Субъект 1: Целевая аудитория (инженеры ПТО, специалисты по ИД)**
- Ищут решения рабочих проблем
- Читают профильные каналы
- Ценят практическую пользу

**Субъект 2: Авторы смежных каналов**
- Могут дать рекламу/упоминание
- Ищут качественный контент для репостов
- Открыты к взаимопиару

### Гипотезы

**H2.1: Экспертные комментарии привлекут подписчиков**
- ЕСЛИ буду оставлять полезные комментарии в крупных каналах
- ТО люди будут переходить на мой канал
- ПОТОМУ ЧТО увидят экспертизу и захотят больше
- РЕЗУЛЬТАТ: +50-100 подписчиков в месяц

Метрики:
- Количество комментариев в неделю
- Переходы на канал (если можно отследить)
- Прирост подписчиков после активности

Задачи:
- [ ] Найти 5-10 каналов для комментирования
- [ ] Настроить ИИ-агента для подготовки комментариев
- [ ] Комментировать 3-5 раз в неделю
- [ ] Отслеживать прирост

---

**H2.2: Практические шаблоны дадут виральность**
- ЕСЛИ буду публиковать готовые шаблоны (АОСР, расчёты, скрипты)
- ТО люди будут сохранять и пересылать
- ПОТОМУ ЧТО это решает их конкретные проблемы
- РЕЗУЛЬТАТ: органический рост через репосты

Метрики:
- Просмотры постов с шаблонами vs обычных
- Репосты/пересылки
- Комментарии с благодарностями

Задачи:
- [ ] Подготовить 5 полезных шаблонов
- [ ] Публиковать 1 шаблон в неделю
- [ ] Анализировать статистику постов

---

**H2.3: Партнёр ускорит развитие**
- ЕСЛИ найду партнёра (технического или контентного)
- ТО смогу выпускать больше контента и продуктов
- ПОТОМУ ЧТО разделим задачи по сильным сторонам
- РЕЗУЛЬТАТ: x2 скорость развития канала

Метрики:
- Количество постов в месяц (до/после)
- Качество контента (вовлечённость)
- Скорость создания продуктов

Задачи:
- [ ] Поговорить с техническим кандидатом (конец января)
- [ ] Определить формат сотрудничества
- [ ] Запустить совместную работу
- [ ] Оценить результаты через 2 месяца

## Статусы гипотез

| Статус | Описание |
|--------|----------|
| `draft` | Сформулирована, не начата |
| `testing` | Эксперимент идёт |
| `validated` | Подтверждена результатами |
| `rejected` | Опровергнута |
| `paused` | Приостановлена |

## Приоритизация гипотез

При выборе, какую гипотезу тестировать первой:

1. **Impact** — насколько сильно повлияет на цель?
2. **Confidence** — насколько уверен в успехе?
3. **Ease** — насколько легко проверить?

ICE Score = (Impact + Confidence + Ease) / 3

Высокий ICE → тестировать первой.

## Интеграция с Second Brain Processor

При обработке записей проверять сигналы гипотез:

**Паттерны интервенции:**
- «если мы...», «думаю, что...», «гипотеза:», «а что если...»
- «попробуем...», «эксперимент:», «можно попробовать...»

**Паттерны результата:**
- «с X до Y», «увеличить на...», «достичь...», «X%»

**Паттерны причины:**
- «потому что...», «из-за того что...», «поэтому...»

Если 2+ паттерна → флаг как сигнал гипотезы.
