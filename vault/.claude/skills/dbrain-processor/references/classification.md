## Decision Tree

```
Запись содержит...
│
├─ Название объекта или имя заказчика/коллеги? ──> TASK (p1-p2) → Todoist + Работа/
│  (Зюзино, Сокольники, Синельников, Кантемир...)
│
├─ «Срочно» или дедлайн сегодня? ────────────────> TASK (p1) → Todoist
│
├─ Рабочая задача без срочности? ────────────────> TASK (p2-p3) → Todoist
│  (сделать, отправить, согласовать, проверить)
│
├─ Автоматизация или программа для ПТО? ─────────> PROJECT → Мои проекты/
│  (скрипт, ассистент, таблица, оптимизация)
│
├─ Телеграм-канал или курс? ─────────────────────> IDEA или TASK → Телеграм-канал/
│  (пост, подписчики, контент, Лебовски)
│
├─ Подработка или сторонний заказ? ──────────────> TASK/PROJECT → Подработка/
│
├─ Обучение или новый навык? ────────────────────> LEARNING → Обучение/
│  (курс, книга, изучить, слепая печать)
│
├─ Личное или здоровье? ─────────────────────────> TASK или REFLECTION
│  (спина, долг, свадьба, Артём)
│
└─ Идея или размышление? ────────────────────────> IDEA → Мысли/идеи/
```

## Apply Decision Filters

Перед сохранением спроси:
- Сколько времени это займёт и как встроить в график?
- Можно ли это автоматизировать?
- Это приближает к финансовой безопасности (500 т.р./мес)?
- Это помогает стать признанным экспертом в отрасли?
- Это масштабируется (курс, консалтинг, продукт)?

Если да на 2+ вопроса → повысить приоритет.
Если «много времени» + «нельзя автоматизировать» + «не приближает к целям» → отклонить или отложить.

## Anti-Patterns (ИЗБЕГАТЬ)

При обработке записей НЕ делать:
- Абстрактные заметки без конкретных действий
- Размытые задачи типа «подумать о...» (конкретизируй!)
- Дублирование без синтеза (кластеризуй похожие!)
- Списки без приоритетов
- Рассуждения без «Следующего действия»

В мышлении избегать:
- Обесценивать свои достижения («это недостаточно»)
- Сравнивать себя с другими в ущерб себе
- Браться за всё сразу без оценки времени
